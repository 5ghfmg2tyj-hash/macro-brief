Notes
  • Data is refreshed by scripts/fetch_live.py, scheduled via a launchd LaunchAgent (see scripts/com.macro-brief.fetch.plist). It fetches server-side (Python urllib), so CORS never enters the picture.
  • The fetcher auto-commits & pushes docs/data/live.json, so the deployed GitHub Pages site reflects the freshest data within ~60 seconds of each run.
  • The Reload button re-reads live.json from the server — it does not trigger a fresh fetch. To force a fetch, run ./scripts/fetch_and_push.sh from the repo root.
  • FRED series update once per business day (~4pm ET); intraday quotes come from Yahoo / CoinGecko.
  • OAS spreads (IG, HY) are in basis points — FRED reports them in percent and the fetcher auto-scales.

    About this app

    A static PWA that renders the weekly macro brief's allocation history, live market data, and the full text of each week's note. Installable from the browser on iOS, Android, macOS, Windows, and Linux.

    Install as an app

    Click the Install button in the header to add Macro Brief to your home screen / Dock / Start Menu.

    On iPhone / iPad: tap the Share icon in Safari, then Add to Home Screen. The app opens full-screen with its own icon, same as any native app.

    How it updates

    Each Monday, a scheduled task generates a new weekly brief and appends (or overwrites, within a ±5 day window) a weekly point to data/history.json and data/flows.json, plus a markdown file in briefs/. Committing and pushing to main is the only deploy step — GitHub Pages serves the /docs folder automatically.

    Data sources

    Bundled JSON for historical tilts and flows. Live quotes are fetched by the local cron from:

    • FRED CSV endpoints — Treasury yields, OAS spreads, WTI, gold fix, Fed funds.
    • Yahoo Finance chart API — equity indices and ETFs.
    • Stooq — fallback for daily closes.
    • CoinGecko — BTC spot.

    Flow data

    ETF and MMF flow figures come from published weekly reports (ICI, ETF.com, VettaFi). They are committed to flows.json each week — not fetched live, because the underlying sources don't expose a clean CORS endpoint.

    Offline

    A service worker caches the app shell so Macro Brief opens instantly and works offline. Live data and briefs are re-fetched when online; the last cached copy is shown when offline.

    Install on iPhone / iPad
    1. Tap the Share icon ⬆︎ at the bottom of Safari.
    2. Scroll down and tap Add to Home Screen.
    3. Tap Add. Macro Brief launches full-screen with its own icon.

    Works in Safari only — iOS blocks this feature in Chrome/Firefox/Edge on iPhone.

    Install on Mac
    1. In Safari, open the File menu.
    2. Click Add to Dock…
    3. Optionally rename, then click Add. Macro Brief opens in its own window.

    Requires macOS Sonoma (14) or later with Safari 17+.

    Install this app
    1. Look for the install icon (⊕ or similar) in the browser's address bar and click it.
    2. No icon? Open the browser menu (⋮ or ☰) and choose Install app or Add to home screen.

    The install option appears after your first visit. Try reloading the page if it's missing.
